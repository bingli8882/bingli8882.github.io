python学习笔记之import

随着计算机硬件水平的提升和计算机网络的普及，脚本语言在软件开发业的地位越来越高。这是计算机发展的一个必然趋势。当今的软件开发行业面临了许多难题，比如系统兼容性，开源等等问题。一方面，现在计算机操作系统已经不是以前的某一家独大的局面，Windows，Mac，Ios，Android，以及Linux的各种发行版，都进入了普通用户的家中。那么一个成功软件，就需要开发不同的版本以便于应对各种主流操作系统。另一方面，随着GitHub等开源社区的崛起，软件开源的呼声越来越强烈。因此脚本软件的优势便凸显了出来。

Java如此流行的一个原因就是对系统的兼容性非常高。通常来说，无论什么系统，只要安装了Java虚拟机，就可以运行Java程序。而Java的这一特性其实就是仿照脚本语言而来的。绝大多数的脚本语言都不需要编译或者预编译，执行的时候直接由解释器翻译运行。因此一个脚本语言的程序能否运行在某一系统上，这完全取决于这个系统是不是拥有此类语言的解释器。这与虚拟机的机制是相似的。而另一方面，脚本语言的程序就是它的源代码，所以也就无所谓开源闭源之说。你只要有这个程序就可以直接打开来看。

而且计算机硬件性能的大幅度提升，导致脚本语言的运行速度也有了保障。因此在开发一些并不需要大量运算软件的时候，脚本语言简洁而高效的风格大大的缩短了开发的时间。还是以Java为例，Java的运行效率和C/C++比，永远无法超越，最多也就是无限接近。但是Java在非底层开发的时候越来越受人喜爱，就是因为开发周期段，易于维护。说白了，人力才是最珍惜的，计算机到底进行了多说运算，只要差距不大，谁也不会在乎。同样的，脚本语言比编译语言在这方面的优势更加明朗。因此学习并熟练掌握一门脚本语言势在必行。

好了，言归正传，我接触过的编程语言可是不少，但是我最熟悉的还是Java，总的来说我还是很喜欢Java的，优点我就不多说了。可是最近要开发一个语言处理方面的项目。虽然Java也能干着事，但是自从看了Python的list之后，我发现用Python做Parser，开发效率可以甩Java半条街。举个例子：

list = ['a','b','c']
if something in list:
  #do something

这段代码用Java实现可是不是这么两三行能搞定的。另一方面身边研究AI的哥们都在用Python，虽然我不研究AI，但是不说跟国际接轨，起码要跟兄弟接轨吧。所以我就决定拾起数年前看过两眼的Python。

写Python遇到的第一个可是说的上是问题的难题就是Python的import实在让人恶心啊。我仅仅是想引用一个父目录中的module，其过程的艰辛，然我觉得世界都不美好了。

文件结构如下：

  • root
    • __inin__.py
    • moduleA.py
    • subPackage
      • __init__.py
      • moduleB.py

我想在moduleB里面引用moduleA里面的一个function，网上搜索的获得了很多方法，对于习惯了Java的孩子，我觉得哪一种都不适合我。好吧只能说我不适合Python吧，呵呵。

第一种方法是：

from ..moduleA import func

或者有人说，最好从root开始引用，这样不会存在namespace冲突的问题，哎，还是Java好啊，上次遇到namespace的问题还是6年前研究C++的时候…从root开始引用的话就变成了：

from root.moduleA import func

然后我满心欢喜的运行：

python moduleB.py

结果，呵呵。

ValueError: Attempted relative import in non-package

我以为我是被eclipse和netbeans这些Java神级IDE宠坏了，从terminal直接运行在子目录下运行module怎么能行呢？Java也不行啊，所以经过反思，又从stackoverflow找了一些资料，回忆起我前几年第一次学Python的时候好像遇到过此类问题，当时我是这样解决的：

python -m root.subPackage.moduleB.py

然后也不知道是stackoverflow的答主说的不清楚，还是我的记忆缺失了，结果我又遇到了新问题：

Import by filename is not supported.

然后我早了很久也没有找到解决这个问题的方法，不知道是我的思路太孤僻一开始就选择了罕见的路线，还是这中间出现了什么问题，反正我是无解了。如果谁知道问题出在哪里，可以给我留言，请不吝赐教，谢谢。

然后我只能走另一条比较常规的路线，那就是用sys.path。

import sys
sys.path.append('..')
from moduleA import func

然后直接运行：

python moduleB.py

终于成功了，内牛满面！！！！

一些人说不建议用相对路径，因为相对路径不稳定，不如使用绝对路径，也就是从系统的根目录 /开始，可是一方面我自己的程序不会动态删module，另一方面还是要保证可移植性。我不可能一直在家里开发，所以决定还是使用相对路径。不过不管怎么说，这个问题总算是解决了。

（PS：打电话给某只一直使用python研究AI的哥们，问这问题的解决办法的时候，得知那孩子从来没有遇到过此类问题。原因是他一直吧所有的code都放在一个module里面……也许才是python的正确使用方法吧，用惯了Java的我一时无法接受……）
