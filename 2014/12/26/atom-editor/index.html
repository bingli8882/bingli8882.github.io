GitHub的Atom编辑器

突然发现我居然是editor和IDE控。我的电脑里面居然有Eclipse，Netbeans，Vim，Kate，Gedit等等一大堆编辑器。那些专用的像python的spyder什么的就不说了，只多不少。以至于当我想编辑一个文件的时候右键弹出来的选择列表长度让我自己都有点小差异。可是在这样的情况下，我今天居然又装了一个编辑器，那就是GitHub开发的Atom。

这算是好奇心使然吧。昨天无意间在GitHub的网站上看到了Atom，广告语挺牛逼的.

A hackable text editor for the 21st Century.

于是乎我就下载了下来，正好有deb安装包，双击安装简单轻松。

这玩意儿看起来很像Sublime，当然我没有用过Sublime，只是去过他的主页看过演示动画。听说业内人士挺推崇Sublime的，但是70美金的价格在我与这个软件之间划下了短期内不可逾越的鸿沟……人穷就不好任性。主要是现在用Vim啥的用的挺开心的，呵呵。（借口！一切都是借口！不要跟我说Sublime怎么怎么好，我不听！我不听！）网上有很多人拿Atom和Sublime去比较，然后得出各种结论，总结来说就是目前Atom还很年轻，但是未来前途无限啥的。我觉得在GitHub这个大框架下，Atom确实很有前景。开源+GitHub这个平台的力量还是非常强大的。不过听说Atom未来会收费……呵呵……希望不会再在我面前划一个沟。

看着那些大神们，能从一个平台，一个软件，或者一套编程语言诞生的时候就开始使用，然后鉴证着它的成长，我无比向往。希望有一天我也能如此，所以当听说这个很有前途而又刚刚诞生的软件时，我还是有尝试并坚持使用它的动力的。于是乎我就让他住进了我的电脑，然后开始了研究和探索的旅程。

说实话，Atom不愧是开发阶段的软件，官方的Document介绍的绝大多说是怎么开发插件和主题，介绍具体怎么使用的只有那么可怜的一页，还是介绍几个常用快捷键的。那一页内用可以总结为一句话：想知道怎么用吗？自己ctrl+shift+p查去。嗯，我承认这个组合键对于Atom很重要，但是这样说也太不User Friendly啊。看看人家Vim，读一遍官方的使用说明基本上就上手了！这样看来我到是希望他们开始收费，那样的话他们一定会更快加的把精力投入到帮助说明文件的建设上。不过现在吗，只能自己边用边摸索了。

打开Atom之后让我惊喜的是，他居然是一个不错的Markdown IDE，我这两天正愁这事呢。没找到什么好用的可以边写边看效果的编辑器。Ubuntu 软件库里面的那两个都是残次品，最后只能退而求其次用一些在线的编辑器。现在好了有了Atom，以后写博客可就方便多了。这算是我这半天没有白忙活，起码有点收获，哈哈！
